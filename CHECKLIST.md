# 잠든 서고 개발 체크리스트

마지막 업데이트: 2026-03-30

이 문서는 현재 구현 상태, 중간 점검 결과, 다음 우선순위를 한 번에 보기 위한 작업 목록이다.

## 현재 위치

- [x] FastAPI 백엔드와 정적 프론트엔드가 연결되어 있다.
- [x] `룬 초안 -> 카논 피드백 / 에코 코멘트 -> 룬 최종본` 루프가 동작한다.
- [x] 설정 문서 3종 편집이 가능하다.
- [x] 루프 상태, 히스토리, Sil 로그, 방문 메타가 로컬 JSON에 저장된다.
- [x] 비용 추정 UI가 있다.
- [x] Gemini API 키가 없을 때 mock 모드로 전체 흐름을 검증할 수 있다.
- [x] 프롬프트 버전 메타가 루프와 히스토리에 저장된다.
- [x] 메인 화면이 채팅 중심 UI로 재구성되어 있다.
- [x] `Workroom` / `Archive` 화면 분리가 끝났다.
- [x] 초안, 피드백, 코멘트, 최종본을 `.md` 첨부처럼 열어볼 수 있다.
- [x] 히스토리 검색, 필터, 상세 보기, export가 된다.
- [x] 카논 피드백이 구조화되어 저장/표시된다.
- [x] 에코 코멘트가 `반응 / 몰입 / 이탈감` 축으로 구조화되어 저장/표시된다.
- [x] 조기 종료 판단 로직이 구조화된 품질 신호를 기준으로 동작한다.
- [x] Windows 실행 파일(`.exe`)과 바로가기가 있다.

- [x] 프론트엔드 최소 상호작용 smoke test 자동화
- [x] 메인 채팅 밀도와 말풍선 정보량 추가 조정
- [x] 첫 실행 / 복귀 / 장기 부재 문구 분기 고도화

- [x] 카논 피드백 구조화

## 중간 점검

- [x] 현재까지의 업데이트는 최초 목적에서 벗어나지 않았다.
- [x] 채팅형 메인 화면, Workroom/Archive 분리, 구조화된 카논/에코 피드백은 모두 `루프를 더 잘 돌리고 비교하기 쉽게 만드는 방향`이다.
- [x] 시각 연출보다 `루프 품질 판단`, `문맥 관리`, `반복 지적의 재활용`이 이제 더 중요한 단계로 넘어왔다.
- [ ] 다음 단계부터는 연출 확장보다 `더 잘 쓰게 돕는 기능`을 우선한다.
- [ ] 다음 단계의 판단 기준은 `이번 루프를 더 잘 끝내게 돕는가`와 `다음 세션에서 더 쉽게 이어 쓰게 하는가` 두 가지로 둔다.
